# 🚀 边缘视觉服务使用指南（语音触发模式）

## 🎯 功能概述

**边缘视觉服务** 让 LeLamp 从"被动工具"变成"主动助手"：

- ✅ **语音触发检测** → 不占用摄像头资源，按需检测
- ✅ **即时LED反馈** → 检测过程可视化
- ✅ **自动手势响应** → 检测到手势自动点头/开关灯等
- ✅ **低功耗设计** → 仅在用户说话时使用摄像头

## 🆕 新增功能

### 1. 语音触发的手势检测

**通过语音命令触发检测，系统提供即时反馈：**

| 语音命令 | 功能 | 反馈 |
|----------|------|------|
| **"检测手势"** | 检测当前手势 | LED闪烁蓝色 → 检测 → 自动响应 |
| **"做了什么手势"** | 识别手势 | LED闪烁蓝色 → 语音播报结果 |
| **"检查一下"** | 综合检查 | 同时检测用户和手势 |

### 2. 自动手势响应

**检测到手势时系统自动做出反应：**

| 手势 | 自动响应 |
|------|---------|
| **👍 点赞** | 点头认可 |
| **👎 踩** | 摇头否认 |
| **✌️ 耶** | 兴奋动作 |
| **👋 挥手** | 开关灯（智能切换） |
| **✊ 握拳** | 停止当前动作 |
| **👆 指向** | 询问指令 |

### 3. 即时LED反馈

**检测过程中的LED变化：**

- **检测前**：LED 闪烁蓝色 (0, 140, 255)
- **检测中**：保持蓝色，提示正在处理
- **检测后**：恢复白色 (255, 255, 255)

## 💬 使用方式

### 方式 1: 语音控制（推荐）

**通过语音命令触发视觉检测：**

```bash
# 手势检测
"检测手势" → 检测当前手势并自动响应
"做了什么手势" → 识别手势

# 快速检查
"检查一下" → 同时检测用户和手势
"看看怎么样了" → 综合状态检查
"扫描一下" → 快速扫描

# 物体识别
"这是什么" → 快速识别物体
"看到什么了" → 描述当前画面

# 用户检测
"有人在吗" → 检测用户在场
"我在吗" → 确认用户在场

# 系统状态
"边缘视觉状态" → 查看服务统计
```

### 方式 2: API 控制

**通过 API 精确控制视觉功能：**

```bash

curl http://192.168.0.106:8000/agent/functions/detect_gesture

# 快速检查
curl http://192.168.0.106:8000/agent/functions/quick_check

# 物体识别
curl http://192.168.0.106:8000/agent/functions/quick_identify

# 用户在场检测
curl http://192.168.0.106:8000/agent/functions/check_presence

# 服务统计
curl http://192.168.0.106:8000/agent/functions/get_edge_vision_stats
```

## ⚡ 性能优势

### 语音触发 vs 后台监听

**语音触发模式的优势：**

| 特性 | 语音触发模式 | 后台监听模式 |
|------|------------|------------|
| **摄像头占用** | ✅ 仅在检测时占用 | ❌ 持续占用 |
| **CPU使用** | ✅ 按需使用，低功耗 | ❌ 持续使用 |
| **响应速度** | ✅ 即时响应 | ⚠️ 需要等待监听周期 |
| **系统稳定性** | ✅ 无资源竞争 | ⚠️ 可能与其他功能冲突 |
| **用户体验** | ✅ 明确知道何时检测 | ⚠️ 不确定何时响应 |
| **适用场景** | ✅ Console 模式 | ❌ 需要远程服务 |

## 🎯 与手动模式的对比

| 功能 | 手动模式 | 语音触发模式 |
|------|---------|------------|
| **触发方式** | 用户说话"检测手势" | 用户说话"检测手势" |
| **响应速度** | 询问后检测 | 询问后检测 + LED反馈 |
| **用户体验** | 需要主动询问 | 即时LED反馈 + 自动动作 |
| **CPU占用** | 按需使用 | 按需使用（相同） |
| **使用场景** | 复杂查询 | 日常交互 |

## 💡 最佳实践

- 语音命令简单直观

## 🎉 开始使用

**语音触发模式已集成到系统中，无需额外配置！**

1. **确保边缘视觉已启用**：
   ```bash
   # 在树莓派的 .env 文件中
   LELAMP_EDGE_VISION_ENABLED=true
   ```

2. **重启服务后直接使用语音命令**
3. **系统会提供LED和动作反馈**

**现在你的 LeLamp 是一个智能语音触发的助手，不会后台占用摄像头资源！** 🚀

---

**注意：本项目不使用 LiveKit 远程服务功能，因此主动监听服务已禁用以避免占用摄像头资源。所有视觉功能都通过语音命令触发，提供最佳的用户体验和系统性能。**
