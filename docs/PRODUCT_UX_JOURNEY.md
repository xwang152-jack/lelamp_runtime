# LeLamp App 功能优先级矩阵与用户旅程

## 📊 功能优先级矩阵 (Impact vs Effort)

```
高影响
  ↑
  │  ┌─────────────┐         ┌─────────────┐
  │  │ 用户账号体系 │  P0     │ 视觉功能入口 │  P1
  │  │ 设备绑定    │         │ 动画按钮面板 │
  │  └─────────────┘         └─────────────┘
  │
  │  ┌─────────────┐         ┌─────────────┐
  │  │ 隐私仪表盘  │  P1     │ 关节控制面板 │  P2
  │  │ Toast通知   │         │ 局域网直连  │
  │  └─────────────┘         └─────────────┘
  │
低影响
  └─────────────────────────────────────────→
     低工作量                      高工作量
```

**图例**:
- **P0 (Must Have)**: 不做就没法商业化,必须在 MVP 阶段完成
- **P1 (Should Have)**: 显著提升用户体验,Alpha 版本必须有
- **P2 (Nice to Have)**: 锦上添花,可以放到 Beta 或正式版

## 🎯 用户角色建模 (Persona)

### Persona 1: 小雨妈妈 (核心用户)

**基本信息**:
- 年龄: 32 岁
- 职业: 互联网公司产品经理
- 家庭: 一个 8 岁的儿子 (小学 3 年级)
- 收入: 年薪 40 万 (中高收入)

**使用场景**:
- 工作日晚上 7-9 点: 辅导孩子作业
- 周末: 陪孩子玩耍,用台灯拍照记录成长
- 出差时: 远程查看孩子学习情况

**需求**:
1. **省时间**: 作业检查要快,不能让我等超过 10 秒
2. **够智能**: 要能真的检查出错题,不是摆设
3. **保护隐私**: 摄像头要有物理开关或明显指示灯
4. **颜值高**: 台灯要好看,放客厅不突兀

**付费意愿**:
- ✅ 愿意为 "远程视频" 功能付费 (¥19.9/月)
- ✅ 愿意为 "云端存储作业照片" 付费 (¥9.9/月)
- ❌ 不愿意为 "更多灯效" 付费 (觉得没必要)

**引用**:
> "我希望台灯能像一个靠谱的家教老师,而不是一个玩具。"

### Persona 2: 豆豆 (次要用户 - 儿童)

**基本信息**:
- 年龄: 8 岁
- 年级: 小学 3 年级
- 性格: 活泼好动,喜欢机器人

**使用场景**:
- 放学回家: 跟台灯打招呼 "我回来啦!"
- 写作业时: 让台灯播放音乐 / 讲笑话
- 睡觉前: 让台灯变成 "星空灯"

**需求**:
1. **要好玩**: 会动、会说话、会变色
2. **要听话**: 我说什么它就做什么 (不要反驳我)
3. **不要凶**: 做错题不要批评我 (温柔纠正)

**行为特点**:
- ❌ 不会打字,只会语音交互
- ✅ 喜欢按按钮 (图标要可爱)
- ✅ 注意力短暂 (超过 5 秒就会走神)

**引用**:
> "我喜欢它会跳舞,但是它总是说'动作有点频繁了',好烦啊!"

**基本信息**:
- 年龄: 28 岁
- 职业: 嵌入式工程师
- 爱好: 开源硬件、智能家居 DIY

- 研究台灯的技术架构

**付费意愿**:
- ❌ 不愿意订阅会员 (觉得自己能搞定)
- ✅ 愿意购买"开发者套件" (¥299,包含源码 + 技术文档)

**引用**:
> "这个项目太酷了,但是文档还不够详细,我想贡献代码但不知道从哪开始。"

## 📈 核心指标体系 (Metrics)

### 用户获取指标 (Acquisition)
- **下载量**: 目标 10,000 次/月 (Alpha 阶段)
- **转化率**: App Store 页面访问 → 下载 = 25%
- **来源**:
  - 自然搜索 40%
  - 社交媒体 30% (小红书、抖音)
  - 朋友推荐 30%

### 激活指标 (Activation)
- **首次连接成功率**: > 90% (如果低于 85%,说明配网流程有问题)
- **引导完成率**: > 70% (完成 3 次对话)
- **首日留存**: > 60%

### 活跃指标 (Engagement)
- **DAU (日活)**: 目标 3,000 用户 (Alpha 阶段)
- **平均使用时长**: > 15 分钟/天
- **功能使用频率**:
  - 作业检查: 1.2 次/天 (工作日)
  - 视频通话: 0.8 次/天
  - 灯光控制: 2.5 次/天

### 留存指标 (Retention)
- **次日留存**: > 60%
- **7 日留存**: > 40%
- **30 日留存**: > 25%

### 商业化指标 (Revenue)
- **付费转化率**: 免费用户 → 会员 = 8%
- **ARPU (人均收入)**: ¥12/月
- **LTV (用户生命周期价值)**: ¥180 (假设留存 15 个月)
- **CAC (用户获客成本)**: < ¥80 (确保 LTV/CAC > 2)

## 🔍 A/B 测试计划

### 测试 1: 作业检查结果展示方式
| 方案 A (纯文字) | 方案 B (可视化) |
|----------------|----------------|
| "第1题正确,第2题错误..." | 绿色 ✅ 红色 ❌ 标注在照片上 |

**假设**: 方案 B 的用户满意度更高 (目标 > 4.5/5 星)

---

### 测试 2: 配网流程
| 方案 A (蓝牙配网) | 方案 B (扫码配网) |
|------------------|------------------|
| App 搜索蓝牙设备 → 传输 Wi-Fi | 扫二维码 → 跳转 Web 页面 → 输入 Wi-Fi |

**假设**: 方案 A 的成功率更高 (目标 > 90%)
