## 📋 评审结论

### 总体评分: 🟡 6.5/10

- ✅ **技术原型**: 8/10 (LiveKit 集成完善,功能齐全)
- ⚠️ **用户体验**: 5/10 (功能入口不清晰,缺少可视化)
- ❌ **商业就绪度**: 4/10 (无账号体系,无法商业化)
- ⚠️ **安全性**: 5/10 (Token 手动生成,无身份验证)

**结论**: 当前 Web Client 完成了技术验证,但要成为**可商业化的产品**,还需要补齐约 60% 的工作。

## 💡 关键改进建议

### 1. 架构升级: 引入中心化后端

**当前架构问题**:
```
Web Client → LiveKit Cloud → Raspberry Pi
          (没有业务后端,无法管理用户和设备)
```

**目标架构**:
```
Mobile App → 业务后端 (Supabase) → LiveKit Cloud → Raspberry Pi
             ↑
             - 用户认证
             - 设备管理
             - Token 签发
             - 支付系统
             - OTA 管理
```

**技术选型**:
- **后端**: Supabase (PostgreSQL + Auth + Edge Functions)
- **移动端**: Flutter (跨平台,LiveKit SDK 支持良好)
- **部署**: Supabase Cloud (无需管理服务器)

### 3. 安全性强化: 多层防护

#### 设备绑定流程:
```
1. 用户扫描设备底部二维码
   ↓ (包含 Device ID + 出厂 Secret)
2. App 调用 POST /api/device/bind
   ↓
3. 后端验证 Secret (HMAC-SHA256)
   ↓
4. 创建 User <-> Device 绑定关系
   ↓
5. 返回长期 Device Token (存储在 App)
   ↓
6. 后续连接时用 Device Token 换 LiveKit Token
```

#### 隐私保护:
- **LED 指示灯**: 摄像头激活时红灯呼吸 (已实现,需在 App 中展示)
- **权限控制**: 主账号、家庭成员、临时访客三级权限
- **使用统计**: 记录摄像头启用次数和时长,支持用户查询
